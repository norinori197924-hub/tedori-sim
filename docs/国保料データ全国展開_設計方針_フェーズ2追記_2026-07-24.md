## フェーズ2: 人口順の都道府県展開リスト作成

### データソース
**総務省「住民基本台帳に基づく人口、人口動態及び世帯数」**(令和7年1月1日時点、2025年8月公表)を使用する。

参照ページ(起点): `https://www.soumu.go.jp/menu_neu/menu06_02.html`
- ここから最新の都道府県別人口データのExcel/CSVダウンロードリンクを辿る

選定理由:
- フェーズ1の市町村コードマスタと同じ総務省という出典元で一貫性がある
- 都道府県コード(5桁コードの先頭2桁、`master.json`の`prefectureCode`)と直接紐付けやすい
- 毎年更新される定期公表統計のため、今後の展開作業でも再利用しやすい

### 実装手順
1. 参照ページから最新(令和7年1月1日時点)の都道府県別人口データのダウンロードリンクを特定する
2. Excel/CSVをダウンロードし、`data/raw/population/`に保存する
   - `source.json`にsourceUrl・sourceTitle・asOf(基準日)・fetchedAt(取得日)を記録する(フェーズ1のsource.jsonと同じ形式)
3. `scripts/population/build_prefecture_ranking.py`(新規)を作成し、都道府県コード+人口数のテーブルを生成する
4. 上記データを使って、**残り44都道府県を人口降順に並べた展開順序リスト**を生成する
   - 出力先: `docs/prefecture_rollout_order.json`(または`scripts/kokuho/`配下でもよい。実装時にリポジトリ構成に合わせて判断)
   - 既に自動収集パイプラインを実証済みの3県(福島県・大阪府・宮城県)は、人口順位に関わらず「対応着手済み」であることが分かるようにする

### 展開順序リストのスキーマ案
```json
{
  "description": "人口降順の都道府県展開順序。docs/国保料データ全国展開_設計方針_2026-07-19.md フェーズ2に基づく。",
  "source": "data/raw/population/source.json",
  "order": [
    {
      "rank": 1,
      "prefectureCode": "13",
      "prefecture": "東京都",
      "population": 0,
      "pipelineStatus": "not_started"
    }
  ]
}
```

- `population`: 実データ取得後に埋める(このドキュメント時点ではプレースホルダ)
- `pipelineStatus`の値域:
  - `verified_pilot`: フェーズ2着手前から自動収集パイプラインの実証が済んでいる県(福島県・大阪府・宮城県)
  - `not_started`: 未着手
  - `in_progress`: パイプライン実行中
  - `done`: 収集完了、`index.json`側のverificationStatus付与まで完了

### 検証ステータスとの関係(復習)
フェーズ3で決めた`verificationStatus`(`verified` / `auto_unverified` / `not_available`)は市区町村単位の値。今回追加する`pipelineStatus`は都道府県単位の進捗管理用で、階層が異なる点に注意する。都道府県のパイプラインが`done`になった時点で、その県内の市区町村の`verificationStatus`が個別に`auto_unverified`として`index.json`に追加される、という関係になる。

### 完了条件
- `docs/prefecture_rollout_order.json`(または同等のファイル)が生成され、47都道府県すべてに人口降順の`rank`が付与されている
- 3県(福島・大阪・宮城)が`verified_pilot`として区別されている
- 人口データの出典(`source.json`)が記録されている
