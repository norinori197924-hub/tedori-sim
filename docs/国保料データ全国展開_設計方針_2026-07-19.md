# 国保料データ全国展開 — 設計方針(Claude Code引き継ぎ用)

作成日: 2026-07-19
対象タスク: 未完了タスク1「国保料データの全国展開(現状14/1,741) — 市町村コードマスタ整備を含む」

---

## 2. フェーズ1: 市町村コードマスタ整備

### データソース
- 総務省「全国地方公共団体コード」(5桁団体コード)を正データとする

### スキーマ案

| フィールド | 内容 |
|---|---|
| `code` | 5桁団体コード |
| `prefecture` | 都道府県名 |
| `city_name` | 市区町村名 |
| `city_name_kana` | 読み仮名(入力補完・検索用) |
| `type` | 市/町/村/特別区/政令指定都市の区 等の区分 |
| `parent_code` | 政令指定都市の区の場合、親市のコード |
| `kokuho_status` | 収集進捗フラグ(下記フェーズ3参照) |
| `verification_status` | データ精度ステータス(下記フェーズ3参照) |

### やること
- 既存14自治体+自動収集済み3市をこのマスタに紐付け、進捗管理の起点とする
- 政令指定都市の「区」の扱い(国保料は市単位で算定されるケースが多いため、区は親市コードを参照させる設計にする想定。実装時に要確認)

---

## 3. フェーズ2: 自動収集パイプラインのスケール

### 展開順序
**人口の多い都道府県・市区町村から優先**して展開する。
- 理由: 少ない展開回数でカバーできるユーザー数を最大化できるため
- 具体順は総務省人口統計等を参照し、都道府県単位でリスト化してから着手

### 失敗時のフォールバック
- 自動収集が失敗、または一次資料へのアクセスに問題がある市町村は `kokuho_status = 保留` としてリスト化
- CLAUDE.mdの教訓通り、同じ調査を何度も繰り返しそうな場合は早めに区切りをつけ、次の手段(ブラウザ目視確認 or 保留)に切り替える

---

## 4. フェーズ3: データ品質の扱い方針(重要な決定事項)

**方式: 登録はするが、検証ステータスをデータに持たせてUI側で正直に開示するハイブリッド方式**

### 理由
- 全件を無条件で暫定値公開すると、家計に関わる金額を扱うツールとして信頼性リスクがある
- 逆に「検証完了まで一切非公開」だと展開が止まりがちで、CLAUDE.mdの教訓(同じ検証作業を繰り返しすぎない)にも反する
- そのため、データ自体は自動収集分も登録した上で、検証状況をステータスとして持たせ、UI側で誠実に開示する方式を採用する

### `verification_status` の3値
| 値 | 意味 |
|---|---|
| `verified` | 一次資料で人間が目視確認済み |
| `auto_unverified` | 自動収集のみ、目視未確認 |
| `not_available` | 未収集 |

### UI側の扱い
- `auto_unverified` の市区町村は、結果画面に小さく「自動収集データ・一次資料での再確認前」の注記を表示する(数値自体は隠さない。国保料の算定ロジックは自治体間で共通性が高く、自動収集の的中率は元々高い想定)

### 検証(目視確認して`verified`に昇格)の優先順位
- 人口の多い都道府県・市区町村から(展開順序と揃えて運用をシンプルにする)
- **政令指定都市・県庁所在地など影響ユーザー数が多い自治体は、自動収集後に必ず目視検証を行い、`verified`になるまで"暫定公開"を許さない特別ルールとする**(このクラスだけは精度優先)

---

## 5. フェーズ4: 運用フロー

- 1,741件を一括で扱うとレビュー不能になるため、**都道府県単位でブランチ/PRを分けて段階的にマージ**する
- 各PRには、その県で収集した市区町村の一覧、`verification_status`の内訳、目視検証を行った自治体とその根拠資料を明記する

---

## 6. 次のアクション(Claude Codeへの依頼事項の例)

1. 総務省の全国地方公共団体コード表を取り込み、上記スキーマで市町村コードマスタを作成
2. 既存14自治体+自動収集済み3市を`verified` / `auto_unverified`で分類してマスタに紐付け
3. 人口順の都道府県展開リストを作成
4. `scripts/kokuho/`のパイプラインに`verification_status`出力を組み込む
5. UI側に`auto_unverified`向けの注記表示を追加
