## 1. 概要と目的

年収と暮らしの条件(8項目)を入力すると、所得税・住民税・社会保険料(または国民健康保険料+国民年金)・手取り額を市区町村単位で概算表示する統合シミュレーターサイト。散在する公的ルール(国税庁・厚労省・各自治体の国保料率)を一元化し、既存サイトが分断している「会社員向け手取り計算」と「フリーランス向け国保計算」を1つの体験に統合することで、AdSenseとアフィリエイトによる収益化を目指す。

## 2. ターゲットユーザーと主要な導線

### ターゲット
- 主: フリーランス・自営業・独立検討中の会社員(市区町村差が大きく効く層。会計ソフト等の高単価アフィリエイトと直結)
- 副: 手取りやふるさと納税上限を知りたい会社員全般

### 主要な導線
1. 検索「国民健康保険料 ○○市 計算」「フリーランス 手取り 計算」等 → シミュレーター → 派生出力(ふるさと納税上限等) → アフィリエイト/AdSense
2. 検索「○○市 国民健康保険料 いくら」(長尾) → 市区町村別ページ → シミュレーター本体
3. 検索「フリーランス 会社員 手取り 比較」 → 切替比較ページ → 会計ソフト等アフィリエイト

## 3. ページ構成・サイトマップ

```
/                       トップ = シミュレーター本体
/result                 計算結果(本体+派生出力を1画面に表示。SPAとして/に統合も可)
/compare                会社員⇔フリーランス切替比較
/city/{都道府県}/{市区町村}/   市区町村別ページ×1,747(国保料率・均等割等+その市での簡易試算)
/articles/…             解説記事(「会社員は市で税金がほぼ変わらない」等の誤解解き系)
/about                  サイト概要・運営者情報
/privacy                プライバシーポリシー(AdSense/GA4対応)
/disclaimer             免責事項(概算であること・税務相談ではないこと・出典と年度)
sitemap.xml
```

### 入力項目(2モード)

**かんたん入力(8項目・基本モード)**
1. 年齢
2. 都道府県
3. 市区町村
4. 子供の数
5. 子供の年齢(各人)
6. 雇用形態(会社員 / フリーランス・自営業。将来拡張: パート、無職・退職)
7. 額面の年収(昨年)
8. 配偶者の有無・配偶者の年収(**任意入力**。未入力時は配偶者なしとして計算)

**詳細入力(かんたん8項目+以下の追加項目。未入力の追加項目はすべて「なし」として計算)**

共通:
- iDeCo・企業型DCの年間掛金(小規模企業共済等掛金控除)
- 生命保険料・地震保険料の年間支払額(生命保険料控除・地震保険料控除)
- 医療費の年間支払額(医療費控除の目安)
- 住宅ローン控除の有無・年末残高(住宅借入金等特別控除)

フリーランスのみ:
- 経費率または経費額
- 青色申告の有無と控除区分(65万 / 55万 / 10万)
- 小規模企業共済の年間掛金

会社員のみ:
- ボーナスの回数・年収に占める比率(社会保険料の按分精度向上)

モード切替はトップページ内のタブまたはアコーディオンで実現し、URLは分けない。

### 出力項目
- 基本: 所得税 / 住民税(所得割+均等割+森林環境税) / 社会保険料の内訳(健康保険・厚生年金・雇用保険、またはお住まいの市区町村の国保料+国民年金) / 手取り額(年・月)
- 派生1: ふるさと納税の控除上限額の目安
- 派生2: 児童手当の受給額目安・住民税非課税判定
- 派生3: 会社員⇔フリーランス切替比較(同一入力を両計算エンジンに通して並列表示。かんたん入力時はフリーランス側を「年収=事業所得」の簡易計算、詳細入力時は経費・青色申告控除を反映)
- 全出力に「概算」表記、計算根拠の年度、出典リンクを付す

## 4. デザイン方針

- トーン: 数字が主役のクリーン系。chosa-rating / hojokin-radar と同系の「公的データを正直に見せる」トーン
- 「あなたの場合、市区町村で変わるのは年○円」のように、俗説を数字で正す編集方針をUIに反映
- 出典・年度・最終更新日を全数値の近傍に明示(信頼性=このサイトの商品)
- モックアップ: 本仕様の承認後にトップページ(入力フォーム+結果表示)のHTMLプレビューを作成し、確認を得る
- 参考UI: 自治体公式シミュレーター(江戸川区等)の入力構成、kokuho-keisan.com の市区町村ナビゲーション(反面教師としてUIの古さを避ける)

## 5. 技術スタックと制約

- ホスティング: GitHub Pages(norinori197924-hub 配下、静的サイト)
- 計算: 全てクライアントサイドJS。料率・税率はJSONデータとしてリポジトリに保持(サーバー・DB不要)
- データパイプライン: GitHub Actions
  - 国保料率(1,747市区町村 × 所得割率/均等割額/平等割額/賦課限度額 × 医療分/支援分/介護分): 都道府県公表の市町村別料率一覧(Excel/PDF)+厚労省資料を一次ソースとし、収集→パース→JSON化。取得できない自治体は「データ未整備」として明示し、都道府県平均等で代替しない
  - 税率・控除(国税庁)、協会けんぽ都道府県別料率、国民年金額、児童手当: 年次改定時に更新
  - データ整備は段階公開: 料率確定済みの自治体から順次「対応済み」とし、カバレッジ(○○○/1,747)をサイト上に表示
- 市区町村別ページ×1,747はビルド時に自動生成(Python)
- 分析・収益: GA4新規プロパティ、AdSense(ca-pub-9507761841542746)、アフィリエイト(ふるさと納税ポータル・会計ソフト・証券口座等。ASP各社の掲載条件確認の上で導入)
- 法令・表示対応: ステマ規制対応のPR表記 / YMYL対応として免責・出典・年度を全ページ徹底 / 金融商品について断定的表現を用いない
- 開発: Windows + Git Bash / Claude Code。SPEC.md承認 → 実装計画承認 → 着手のフローを厳守
- リリース方針: **同時設計・段階リリース**。計算エンジン・データ構造・UIは2モード前提で設計するが、公開はかんたん入力モードを第1弾、詳細入力モードを第2弾とする。第1弾の時点で詳細モードの入力欄は「準備中」として非表示またはグレーアウト

## 6. スコープ外(初期リリースでやらないこと)

- 保育料の算出(自治体別料金表の整備コスト大。将来拡張候補として保留)
- 副業収入・株式等の譲渡所得・配当所得・不動産所得の反映(詳細入力でも対象外。計算分岐の複雑化とYMYLリスクに対して需要が見合わないため)
- パート・アルバイトの「年収の壁」詳細判定(配偶者年収は控除計算にのみ使用)
- 住民税の自治体別例外(名古屋市減税・一部超過課税)の個別対応(初期は標準税率で統一し、注記で対応)
- 国保組合(建設国保・文芸美術国保等)への対応
- 住民税非課税「世帯」の厳密判定(本人の非課税判定のみ)
- 過年度の計算(最新年度のみ)
- 英語対応・アプリ化

## 7. 完了条件(検証可能な形で)

1. 8項目の入力から、会社員・フリーランス両方の手取り概算が表示される
2. 国保料率データのカバレッジが全1,747市区町村に到達している(未整備自治体は「未整備」表示があり、無言の欠落がない)
3. 検算: 自治体公式シミュレーター(江戸川区等、3自治体以上)と同条件で比較し、国保料の乖離が±1%以内
4. 検算: 国税庁の源泉徴収税額・住民税の標準計算例と照合し、所得税・住民税が一致する
5. ふるさと納税上限額が大手ポータルのシミュレーター結果と概ね一致する(±2,000円以内)
6. 市区町村別ページ1,747件が自動生成され、sitemap.xmlに全件含まれる
7. スマホ表示(375px幅)で入力フォーム・結果表示が崩れない
8. 免責事項・プライバシーポリシー・運営者情報・PR表記が全ページから到達可能
9. GitHub Actionsの料率更新ワークフローが手動実行で成功し、live siteへの反映まで独立検証で確認できる(push → Actions → live site)
10. かんたん入力のみで計算が完結する(詳細項目が未入力でもエラーにならず、「なし」として計算される)
11. 詳細入力時、国税庁の確定申告書等作成コーナーの計算例と照合し、主要控除(iDeCo・生命保険料・青色申告控除・住宅ローン控除)の反映額が一致する

※ 完了条件1〜10は第1弾(かんたん入力)リリースの条件、11は第2弾(詳細入力)リリースの条件とする。
